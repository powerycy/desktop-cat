# 关键流程

## Flow 1: 设置专注与休息节奏

| 项 | 内容 |
| --- | --- |
| Actor | 用户 |
| Precondition | 应用已启动，设置页可打开 |
| Success outcome | 工作时长、休息时长、休息模式、开机启动、皮肤缩放保存成功 |

步骤：

1. 用户在 `SettingsView.vue` 修改配置。
2. 前端调用 Tauri `save_config`。
3. Rust 命令写入 Tauri store，并同步开机启动状态。
4. 运行中的计时器状态热更新。

授权与边界：

- 无账号角色系统，所有设置都属于当前本机用户。
- 页面到 Tauri 是主要信任边界。
- 失败情况应展示保存错误，不应吞掉配置写入失败。

## Flow 2: AI 生成桌宠皮肤

| 项 | 内容 |
| --- | --- |
| Actor | 用户 |
| Precondition | 用户输入 Gemini API Key、皮肤名称、角色描述 |
| Success outcome | 生成的 PNG 动作帧保存到本机，并成为当前皮肤 |

步骤：

1. 用户在设置页输入 API Key 和提示词。
2. 前端调用 `generate_ai_sprite_set`。
3. Rust 校验皮肤名称和提示词。
4. Rust 调用 Gemini 图片接口。
5. 返回图片被裁切成 `walk`、`sit`、`sleep`、`rest` 帧。
6. 生成 `manifest.json`，保存到应用数据目录。
7. 前端刷新皮肤列表并保存当前选择。

授权与边界：

- API Key 不在源码中硬编码。
- 用户提示词和 API Key 会离开本机发往 Google。
- 皮肤名称只允许字母、数字、`-`、`_` 或非 ASCII 字符，避免路径穿越。

## Flow 3: 强制休息

| 项 | 内容 |
| --- | --- |
| Actor | 用户、计时器 |
| Precondition | 专注计时结束，或连续跳过达到阈值 |
| Success outcome | 全屏休息窗口打开，倒计时结束后记录一次完成休息 |

步骤：

1. 计时器到达工作时长。
2. 用户可以选择休息或跳过一次。
3. 连续跳过达到阈值后，下一次进入强制休息。
4. Tauri 创建 `rest` 窗口并置顶。
5. `RestView.vue` 倒计时并记录完成、跳过或中断结果。

授权与边界：

- 无远程权限检查，属于本机健康提醒逻辑。
- 全屏窗口是对本机交互的强干预，应在产品说明中清楚告知。

## Flow 4: 自定义 PNG 皮肤导入

| 项 | 内容 |
| --- | --- |
| Actor | 用户 |
| Precondition | 用户选择必需动作帧 PNG |
| Success outcome | 皮肤文件保存到应用数据目录并可选择 |

步骤：

1. 用户选择动作帧文件。
2. 前端检查必需动作是否齐全。
3. 文件转 base64 传给 Tauri。
4. Rust 写入皮肤目录并生成 manifest。

授权与边界：

- 文件读取由用户通过文件选择器触发。
- 写入范围应限制在应用数据目录。
